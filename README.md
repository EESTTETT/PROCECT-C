# Отчет о реализации проекта кластеризации K-Means

Данный проект представляет собой высокопроизводительную гибридную систему кластеризации, где вычислительное ядро и алгоритмы внутренней оценки реализованы на языке C, а среда Python используется исключительно для генерации начального эталонного датасета и графического визуализации

---

## 1. Структура проекта и назначение файлов

Проект построен по модульному принципу, что обеспечивает четкое разделение ответственности (Separation of Concerns) и исключает дублирование логики:

* **`kmeans.h` / `kmeans.c**` — Модуль алгоритма кластеризации. Содержит реализацию итерационного процесса классического K-means , а также функцию умного вероятностного выбора начальных центроидов K-means++.
* **`metrics.h` / `metrics.c**` — Модуль внутренней оценки качества. Реализует математический аппарат Силуэтного анализа (Silhouette Analysis) для каждой точки датасета с вычислительной сложностью **O(N²)** для автономного поиска оптимального числа кластеров.
* **`dataset.h` / `dataset.c**` — Модуль ввода-вывода (I/O). Отвечает за парсинг входного CSV-файла, замену точек на запятые для локализации данных и экспорт результатов вычислений.
* **`main.c`** — Главный диспетчер вычислительного ядра на C. Координирует загрузку данных, запускает исследовательский цикл подбора **k** (от 2 до 10), выводит подробный лог метрик в консоль, автоматически выбирает оптимальную структуру по максимуму силуэтного коэффициента и сохраняет финальную разметку вместе с координатами центроидов.
* **`test_runner_new.py`** — Вспомогательный скрипт на Python. Отвечает за генерацию синтетического датасета с заранее известной кластерной структурой (используется инструмент `make_blobs` из библиотеки `scikit-learn`). Скрипт работает в гибридном режиме: после генерации данных он приостанавливает работу, полностью уступая управление независимому вычислительному ядру на C. После завершения расчетов скрипт считывает результаты, рассчитывает индекс точности (ARI) и строит цветной интерактивный график.
* **`Makefile`** — Скрипт автоматизации сборки. Объединяет компиляцию всех модулей C и запуск готового бинарного файла в одну короткую команду.

---

## 2. Цикл программы

Жизненный цикл эксперимента разделен на три фазы: подготовка данных (эталонного) в Python, независимые вычисления в C-файлах и финальный графический анализ. Система последовательно проходит следующие этапы:

1. **Этап Инициализации и Подготовки данных [Py]:** При запуске `test_runner.py` функция `make_blobs` генерирует случайное распределение из 2000 точек (размер настраивается), сгруппированных в 3 облака. Эти данные вместе с истинными метками записываются в промежуточный файл `synthetic_input.csv`. После этого скрипт переходит в режим ожидания (паузы).
2. **Этап Автоматической Сборки и Запуска [Makefile]:** В отдельном терминале вызывается утилита `make`. Система автоматически запускает компилятор `gcc`, собирает `.c` файлы с подключением математической библиотеки `-lm`, создает исполняемый файл `kmeans_analyzer` и отправляет его на выполнение.
3. **Этап Исследовательского Анализа [C]:** Программа на C загружает координаты. Запускается изолированный цикл перебора числа потенциальных кластеров **k** от 2 до 10. Для каждого **k** отрабатывает алгоритм K-means++, рассчитывается инерция (WCSS) и средний силуэтный коэффициент. Процесс логируется в консоль.
4. **Этап Автоматического Принятия Решения [C]:** Программа автономно анализирует историю метрик. Значение **k**, при котором силуэт достиг глобального максимума, признается оптимальным (в данном случае **k = 3**). Координаты финальных центроидов выводятся на экран.
5. **Этап Финального Экспорта результатов [C]:** Выполняется финальный прогон алгоритма K-means для найденного оптимального **k**. Координаты точек, эталонные метки и предсказанные C-кодом кластеры сохраняются в `clusters_optimal.csv`, а история подбора — в `metrics_output.csv`. Ядро C завершает работу.
6. **Этап Валидации и Отрисовки [Py]:** Разработчик возвращается к окну Python и нажимает Enter. Скрипт выходит из режима ожидания, считывает результаты, рассчитывает индекс ARI для верификации и открывает интерактивное окно, где каждая из 2000 точек окрашена в цвет кластера, определенного C-ядром.

## 3. Инструкция по запуску проекта

### Шаг 3.1. Установка необходимых библиотек

Убедитесь, что в интерпретаторе Python установлены пакеты для анализа данных и визуализации:
```
/usr/local/bin/python3 -m pip install numpy scikit-learn matplotlib

```

### Шаг 3.2. Запуск интеграционного теста

1. **В Терминале №1** запустите скрипт генерации датасета:
```
/usr/local/bin/python3 test_runner.py

```
или просто запускаем файл `test_runner_new.py`


2. **В Терминале №2** соберите и запустите ядро C одной командой через Makefile:
```
make

```

*(В консоли отобразится лог расчетов, подбор инерции, силуэтов и итоговые центроиды).*

3. **В Терминале №1** вернитесь к окну Python и нажмите клавишу **Enter** для вывода графиков.

## 5. Описание примененных методов и алгоритмов

* **Алгоритм K-Means (Метод k-средних):** Базовый алгоритм кластеризации, основанный на чередовании двух шагов. *Шаг ожидания (E-шаг):* привязка каждой точки к ближайшему центроиду по Евклидову расстоянию. *Шаг максимизации (M-шаг):* пересчет координат центроида как центра масс всех точек кластера. Цикл повторяется до стабилизации меток.
* **Метод K-Means++:** Модификация инициализации центроидов. Первый центр выбирается случайно, а каждый следующий — с вероятностью, пропорциональной квадрату расстояния от точки до существующих центров. Это исключает неудачный старт и ускоряет сходимость.
* **Силуэтный анализ (Silhouette Score):** Внутренняя метрика оценки качества. Для каждой точки вычисляется значение **a** (среднее расстояние до точек своего кластера) и **b** (среднее расстояние до точек ближайшего чужого кластера).Среднее значение по всему датасету показывает качество разбиения (от -1 до 1). Глобальный максимум указывает на идеальное количество кластеров.
* **Adjusted Rand Index (ARI):** Внешняя метрика сходства двух разбиений. Оценивает, насколько предсказанные кластеры соответствуют эталону. Значение `1.0` означает полное совпадение структур, а `0.0` указывает на случайное угадывание. ARI учитывает именно группировку пар элементов, игнорируя перестановку названий самих меток (например, класс `0` стал классом `2`).
* **Инструмент `make_blobs`:** Функция из библиотеки `scikit-learn` для генерации изотропных облаков данных. Позволяет гибко настраивать количество точек и дисперсию для создания строго контролируемой тестовой среды.
