# Отчет о реализации проекта автоматической кластеризации K-Means

Данный проект представляет собой высокопроизводительную систему кластеризации
---
## 1. Структура проекта и назначение файлов

Проект построен по модульному принципу, что обеспечивает разделение ответственности (Separation of Concerns) и упрощает поддержку кода

* `kmeans.h` / `kmeans.c` — Модуль алгоритма кластеризации. Содержит реализацию итерационного процесса классического K-means (E-шаг и M-шаг), а также функцию умного вероятностного выбора начальных центроидов K-means++.
* `metrics.h` / `metrics.c` — Модуль внутренней оценки качества. Реализует математический аппарат **Силуэтного анализа (Silhouette Analysis)** для каждой точки датасета с вычислительной сложностью $O(N^2)$ для автономного поиска оптимального числа кластеров.
* `dataset.h` / `dataset.c` — Модуль ввода-вывода (I/O). Отвечает за парсинг входного CSV-файла и экспорт результатов. Содержит функцию локализации числовых данных 
* `main.c` — Главный диспетчер вычислительного ядра на C. Координирует загрузку данных, запускает исследовательский цикл подбора $k$ (от 2 до 10), автоматически выбирает оптимальную структуру по максимуму силуэтного коэффициента и сохраняет финальную разметку.
* `test_runner.py` — Интеграционный скрипт верхнего уровня на Python. Автоматизирует весь жизненный цикл эксперимента: генерирует облака точек, компилирует C-код, запускает бинарный файл, верифицирует точность через внешние метрики и строит финальный интерактивный график для визуализации

---

## 2. Этапы работы кода (Жизненный цикл программы)

При запуске управляющего скрипта `test_runner.py` система последовательно проходит следующие этапы:

1. **Этап Инициализации и Генерации [Py]:** Функция `make_blobs` генерирует случайное распределение из 2000 точек(или других выставленных значений), сгруппированных в 3 гауссовских облака. Эти данные вместе с истинными метками записываются в промежуточный файл `synthetic_input.csv`.
2. **Этап Автоматической Компиляции:** cкрипт через подпроцесс вызывает компилятор `gcc`, собирая все `.c` файлы проекта с флагом оптимизации математики `-lm` в единый исполняемый файл `kmeans_analyzer`.
3. **Этап Исследовательского Анализа (Ядро C):** Исполняемый файл загружает координаты. Запускается цикл перебора числа потенциальных кластеров $k$ от 2 до 10. Для каждого $k$ отрабатывает алгоритм K-means и рассчитывается средний силуэтный коэффициент датасета.
4. **Этап Автоматического Принятия Решения [C]:** Программа на C анализирует полученную историю метрик. Значение $k$, при котором силуэт достиг глобального максимума, признается оптимальным архитектурным решением (в нашем случае $k=3$).
5. **Этап Финального Экспорта [C]:** Выполняется финальный прогон K-means для оптимального $k$. Координаты точек, эталонные метки и предсказанные C-кодом кластеры сохраняются в `clusters_optimal.csv`. История подбора сохраняется в `metrics_output.csv`.
6. **Этап Валидации и Oтрисовки [Py]:** Python считывает результаты из C, рассчитывает индекс ARI для проверки качества и открывает интерактивное окно с цветным графиком, где каждая точка окрашена в цвет кластера, определенного C-ядром.

## 4. Как увидеть и интерпретировать результаты

После завершения вычислений результаты можно изучить в трех представлениях:
1. **Графический отчет (Всплывающее окно):** На экране автоматически откроется окно Matplotlib с графиком scatter-plot. Вы увидите 2000 точек или столько сколько выставлено в коде (const int n_points -- в main.c и так же n_samples -- в test_runner.py). Геометрически близкие точки будут окрашены в один цвет. Четкое разделение на цветные группы без перемешивания подтверждает корректность работы алгоритма


2. **В Консоли (Терминале):** Вы увидите подробные логи работы C-ядра (значения инерции и силуэта для каждого $k$, координаты найденных центроидов). В самом конце Python выведет статус:
* `Индекс сходства структуры`
* `СТАТУС ТЕСТА`


3. **Локальные файлы результатов (CSV):** В корневой папке проекта появятся таблицы. Благодаря тому, что C-код принудительно заменяет точки на запятые, файл `clusters_optimal.csv` можно напрямую открыть в **Microsoft Excel** или десктопной версии. Вы можете выделить столбцы `x` и `y` и построить нативную точечную диаграмму прямо в Excel — она полностью совпадет с визуальным графиком из Python

## 5. Описание примененных методов и алгоритмов

* **Алгоритм K-Means (Метод k-средних):** Базовый алгоритм кластеризации, основанный на чередовании двух шагов. *Шаг ожидания :* привязка каждой точки к ближайшему центроиду по Евклидову расстоянию. *Шаг максимизации (M-шаг):* пересчет координат центроида как центра масс (среднего арифметического) всех точек, зачисленных в данный кластер. Цикл повторяется до стабилизации меток.
* **Метод K-Means++ (Умный старт):** Модификация инициализации центроидов. Вместо полностью случайного выбора, первый центр выбирается случайно, а каждый следующий — с вероятностью, пропорциональной квадрату расстояния от точки до уже существующих центров. Это исключает неудачный старт (когда центроиды изначально оказываются слишком близко) и ускоряет сходимость алгоритма в разы.
* **Силуэтный анализ (Silhouette Score):** Внутренняя метрика, не требующая знания истинных меток. Для каждой точки вычисляется $a$ (среднее расстояние до точек своего кластера) и $b$ (среднее расстояние до точек ближайшего чужого кластера). Силуэт точки равен $(b - a) / \max(a, b)$. Среднее значение по всему датасету показывает качество разбиения. Диапазон от -1 до 1. Глобальный максимум указывает на идеальное количество кластеров.
* **Adjusted Rand Index (ARI — Скорректированный индекс Ранда):** Внешняя метрика сходства двух разбиений. Используется для верификации. Оценивает, насколько предсказанные кластеры соответствуют эталону от `make_blobs`. ARI = 1.0 означает полное совпадение структур, а значение около 0.0 указывает на случайное угадывание. При этом метка кластера может называться иначе (например, класс `0` в эталоне стал классом `2` в C), но ARI учитывает именно группировку пар элементов, а не их имена.
* **Инструмент `make_blobs`:** Функция из библиотеки scikit-learn, используемая в тестовом модуле для генерации изотропных Гауссовских подвыборок (облаков данных). Позволяет гибко настраивать количество точек, размерность пространства и степень разброса (дисперсию) вокруг центров для создания контролируемой тестовой среды.
